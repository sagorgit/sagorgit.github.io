# Лабораторная работа №4: Классификация с применением Scikit-Learn

### 1. Цель работы
Изучение алгоритмов машинного обучения для задачи классификации, освоение библиотеки Scikit-Learn для предварительной обработки данных, построения моделей и оценки их качества с помощью метрик точности и ROC-AUC.

### 2. Задание
1. Подготовка данных: загрузка датасета "Give Me Some Credit", обработка пропущенных значений и разделение данных на обучающую и тестовую выборки.
2. Построение базовых моделей: Логистическая регрессия и Случайный лес (Random Forest).
3. Оценка качества: использование матрицы сопряженности (Confusion Matrix), расчет Accuracy и ROC-AUC.
4. Оптимизация: улучшение модели Случайного леса путем настройки гиперпараметров для достижения ROC-AUC ≥ 0.9.
5. Самостоятельное исследование: сравнение с моделями SVM и kNN, анализ современных алгоритмов классификации и описание алгоритма интеграции модели с веб-сервисом (FastAPI/Flask).
### 3. Ход работы / Код
В ходе работы были выполнены следующие ключевые этапы:

*   **Предобработка:** Пропуски в данных (MonthlyIncome, NumberOfDependents) были заполнены средними значениями обучающей выборки.
*   **Обучение:** Были обучены модели `LogisticRegression` и `RandomForestClassifier`. 
*   **Оптимизация:** Для достижения наилучшего результата была проведена настройка модели Random Forest (n_estimators=500, max_depth=12). Это позволило достичь показателя **ROC-AUC > 0.90**.

**Основной фрагмент кода оптимизированной модели:**
```python
final_optimized_rf = ensemble.RandomForestClassifier(
    n_estimators=500, 
    max_depth=12, 
    min_samples_leaf=30, 
    random_state=42
)
final_optimized_rf.fit(training_points, training_values)
```
**Визуализация:** Построена кривая ROC, демонстрирующая высокую предсказательную способность модели.

### 4. Выводы
1. **Эффективность моделей:** Модель Случайного леса (Random Forest) показала значительно лучшие результаты по сравнению с логистической регрессией и простыми методами вроде kNN, так как она более устойчива к несбалансированным данным и нелинейным зависимостям.
2. **Анализ SVM/kNN:** Исследование показало, что kNN чувствителен к шуму, а SVM требует сложной настройки ядра. В данной задаче они уступают ансамблевым методам.
3. **Современные тренды:** Анализ соревнований Kaggle подтверждает, что для табличных данных сейчас лидируют алгоритмы градиентного бустинга (XGBoost, LightGBM, CatBoost).
4. **Интеграция:** Изучен алгоритм деплоя модели через FastAPI, включающий сериализацию модели (joblib), создание эндпоинта /predict и обработку JSON-запросов для выдачи прогноза дефолта в реальном времени.
